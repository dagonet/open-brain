# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Open Brain, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, use one of these methods:

1. **GitHub Private Security Advisory** (preferred): Go to the [Security tab](https://github.com/dagonet/open-brain/security/advisories/new) and create a private advisory.
2. **Email**: Contact the maintainer directly via GitHub profile.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- Acknowledgment within 48 hours
- Status update within 7 days
- Fix timeline depends on severity

## Scope

This policy covers:
- The Open Brain codebase (CLI, MCP server, edge functions, web dashboard)
- Database schema and RLS policies
- Authentication and authorization logic

Out of scope:
- Supabase platform vulnerabilities (report to [Supabase](https://supabase.com/security))
- OpenAI API vulnerabilities (report to [OpenAI](https://openai.com/security))
